# IMAGE PROMPT SHEET TEMPLATE
## Long-Form Generation Prompts for Y-It Visual Assets

**Version:** 1.0
**Purpose:** Master template for generating all visual assets for both KDP and Web versions
**Usage:** Copy this template, fill in topic-specific details, generate assets

---

## TEMPLATE INSTRUCTIONS

1. Copy this file to `/archives/02-CONTENT/[topic-slug]/IMAGE_PROMPT_SHEET_[TOPIC].md`
2. Replace all `[TOPIC]` placeholders with your topic
3. Customize archetype names and case study details from your manuscript
4. Generate images using your preferred AI tool (Midjourney, DALL-E, Stable Diffusion)
5. Save generated assets to `/archives/03-DESIGN/[topic-slug]/assets/`

---

## STYLE GUIDE FOR ALL Y-IT IMAGES

**Overall Aesthetic:**
- Modern, clean, slightly satirical
- Color palette: Cream (#F5F2EC), Blue accent (#5D7BEA), Red alert (#E35050), Green success (#6EC972)
- Typography integration: Sans-serif, bold headers
- Mood: Informed skepticism, data-driven, not mean-spirited

**Character Style:**
- Semi-realistic illustration style
- Diverse representation in archetypes
- Expressions showing aspiration, then reality
- No specific real-person likenesses

**Data Visualization Style:**
- Clean, minimal design
- High contrast for readability
- Animated where applicable (web)
- Print-safe color versions (KDP)

# SECTION 1: COVER & HERO IMAGES

## IM-00-COVER / WEB-00-HERO
**Cover Image / Hero Banner**

**KDP Prompt (Print Cover):**
```
Create a 6x9 inch book cover for "[TOPIC TITLE]" in the Y-It nano-book style.

Visual concept: Split composition showing the PROMISE on the left (glamorous, golden, aspirational - [DESCRIBE SPECIFIC PROMISE VISUAL FOR THIS TOPIC]) versus REALITY on the right (mundane, gray, exhausted - [DESCRIBE SPECIFIC REALITY VISUAL FOR THIS TOPIC]).

Style: Modern editorial illustration, clean lines, limited color palette (cream, blue, red accents). Semi-satirical but not cartoonish. Professional publishing quality.

Typography space: Leave clear space at top for title "Y-IT: [TOPIC]" and bottom for subtitle.

Mood: Intelligent skepticism, data-driven, not mean-spirited.

Dimensions: 6" x 9" at 300 DPI, CMYK color space
```

**Web Prompt (Hero Banner):**
```
Create a widescreen hero banner for "[TOPIC TITLE]" web article.

Visual concept: Dramatic split-screen showing PROMISE (left side - [DESCRIBE: laptop on beach, money raining, freedom imagery specific to this topic]) transitioning to REALITY (right side - [DESCRIBE: exhausted person at desk, empty wallet, stress indicators specific to this topic]).

Style: Modern, clean editorial illustration. Y-It brand colors (cream #F5F2EC background, blue #5D7BEA and red #E35050 accents).

Animation-ready: Design with clear layers for potential parallax/reveal animation.

Dimensions: 1920 x 1080 pixels, RGB, web-optimized
```

**Guru Revenue Comparison Chart**

```
Comparison:
```

## IM-08-DECISION-TREE
**Decision Framework Flowchart**

**Prompt:**
```
Create a decision flowchart for "Should I Do [TOPIC]?"

Flowchart structure:
START: "Are you considering [TOPIC]?"
↓
Q1: "Do you have $[X]+ in risk capital you can afford to lose?"
→ NO: "This isn't for you. See alternatives."
→ YES: Continue
↓
Q2: "Can you commit [X] hours/week for [Y] months?"
→ NO: "This isn't for you. See alternatives."
→ YES: Continue
↓
Q3: "Do you have [SPECIFIC SKILL/ADVANTAGE]?"
→ NO: "Your odds are very low. See alternatives."
→ YES: Continue
↓
Q4: "Can you handle [X]% probability of complete loss?"
→ NO: "This isn't for you. See alternatives."
→ YES: Continue
↓
FINAL: "You may proceed with extreme caution. Set quit triggers."

Style: Clean flowchart, color-coded paths (green = proceed, red = stop, yellow = caution)

Dimensions (KDP): 4" x 3" at 300 DPI
Dimensions (Web): 800 x 600 pixels, interactive clickable version
```

# ASSET NAMING CONVENTION

**File Naming Format:**
```
[TOPIC-SLUG]_[MARKER-ID]_[VERSION].[FORMAT]

Examples:
dropshipping_IM-00-COVER_v1.png
dropshipping_WEB-02-ARCH-1_v1.png
dropshipping_IM-06-CASE-1_v2.jpg
```

**Version Control:**
- v1 = Initial generation
- v2 = Refined after review
- v3 = Final approved version

**Folder Structure:**
```
/archives/03-DESIGN/[topic-slug]/assets/
├── /kdp/
```
