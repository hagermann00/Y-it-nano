## ⚡ KEY PRINCIPLES (DON'T FORGET)

1. **Ship Fast > Perfection**
   - 80% polish is good enough
   - Get books live, get feedback, iterate

2. **Feedback Over Assumptions**
   - 1 complaint = note it
   - 3+ complaints = it's a pattern, fix it
   - 0 complaints = ship as-is

3. **Bugs Immediate, Iteration Later**
   - Broken link? Fix in 48 hours
   - "Should have more detail"? Wait for 3+ requests

4. **Tools Second, Execution First**
   - Don't wait for perfect dashboard tool
   - Don't perfect designer workflow before starting
   - Pick something, start, improve later

5. **Validation Gate is Week 13**
   - Not before
   - Don't over-optimize until you know if it works
   - Then double down on winners
